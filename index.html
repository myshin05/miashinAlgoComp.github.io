Blog Post:

Hi, so for the Hamster Cellular Automata, I first created an 8x5 2D grid, I didn't want to make it too big because I wanted to add individual pictures for each of the squares icl. Ok, so bc there were 40 total squares, I decided to do 8 different pentatonic scales where each column is a different scale because I love my pentatonic scales. So I directly inputted the scale patterns to the columnScales array so that each box has its own little note(some are repeats). After that I uploaded all the hamster images to the boxes in js. After that, I decided on four different systems to guide the automata behavior and each of them have a survive and birth rule, in which I specify how many neighbors a box needs to have to either keeping living to start living, if it doesn't meet either criteria it dies. There are also methods for "collecting" every note that is alive at that point to play their respective notes during that generation, and another to actually play the entire grid at once. I had issues with this in the beginning, because it seemed like only new notes would start playing. For each generation, the selected system is applied to the current grid and moves onto the next generation with the rule applied and so on. I also added a speed slider, generation counter, and step incrementer like in the Game of Life, and I also added a random grid generator so we can get out of any empty/stuck grids. Some insights that I gained from programming this beyond what we covered in class is that like it's harder than I thought to come up with certain rules that will kind of prolong the lifetime of the blocks because I kind of wanted certain systems like Hamster Kill to die out quickly and Hamster Love to last long and be filled a lot, so I had to do some experimentation to see what would work best to kind of fit what I had in mind in terms of the general rule/pattern. I had also initally wanted to do like one for each of the 4 classes of cellular automata but, after some experimentation, I was kind of deterred just because it felt really daunting to try to directly fit a certain system into one of these classes. I would say that quite a few of my systems stall/don't change at a certain point or sometimes altogether disappear but it mostly depends on the initial randomized grid, but that can be fun too yay. But, I think the process of coming up with the rules was definitely the most fun and I played around with it a lot. This process also helped me understand the actual process of creating cellular automata and I thought it was super interesting and not as like mystified as it was before.
